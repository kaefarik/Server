# Система мониторинга облачности и управления двигателем

Распределённая система компьютерного зрения для автоматического управления двигателем в зависимости от того, перекрывают ли облака солнце.

## 🏗 Архитектура системы

Система состоит из двух независимых компонентов:

**Client (Vision System)** — запускается на устройстве с камерой или ПК с видеофайлом:
- захватывает кадры через **OpenCV**;
- детектирует облака и солнце через HSV-маску;
- отслеживает облака с помощью **фильтра Калмана** и предсказывает траекторию;
- при угрозе перекрытия отправляет HTTP-команды на сервер.

**Server (Hardware Controller)** — запускается на ARM-устройстве (Raspberry Pi и т.п.):
- принимает HTTP-команды по сети;
- управляет состоянием мотора через **конечный автомат (FSM)**, защищая от дублирующих команд;
- переключает физические пины через **GPIO (ioctl)**.

## 🚀 Сборка

### Требования

- C++17 или выше
- `OpenCV`, `httplib` (подтягивается через FetchContent), `nlohmann/json` (подтягивается через FetchContent)

### Клиент (Vision)

```bash
mkdir build && cd build
cmake ..
make
```

### Сервер (ARM)

Кросс-компиляция внутри Docker-контейнера:

```bash
# Из директории net/
sudo docker run -it --rm -v $(pwd):/workspace ubuntu:22.04 bash

# Внутри контейнера:
apt update && apt install -y g++-arm-linux-gnueabihf
cd /workspace
arm-linux-gnueabihf-g++ \
  -static -pthread \
  -o arm_server \
  src/server.cpp \
  src/gpio_control.c \
  -I./library -I./include
```

## ▶️ Запуск

### Сервер

**На реальном ARM-устройстве** (Raspberry Pi и т.п.):
```bash
scp arm_server pi@<адрес_устройства>:~/
ssh pi@<адрес_устройства>
./arm_server 0.0.0.0 8080
```

**На ПК через QEMU** (для тестирования без железа):
```bash
# Установить QEMU если не установлен
apt install -y qemu-user-static   # Ubuntu/Debian
# или
sudo pacman -S qemu-user-static   # Arch Linux

# Запустить с пробросом порта через Docker
sudo docker run -it --rm -v $(pwd):/workspace -p 8080:8080 ubuntu:22.04 bash

# Внутри контейнера:
apt install -y qemu-user-static
qemu-arm-static ./arm_server 0.0.0.0 8080
```

> ⚠️ Передавай `0.0.0.0` а не конкретный IP — иначе Docker не сможет пробросить порт наружу.

### Клиент

```bash
# С веб-камерой:
./traker -ip <ip_сервера> -p <порт> -c

# С видеофайлом:
./traker -ip <ip_сервера> -p <порт> -v <путь_к_видео>
```

**Примеры:**
```bash
# Сервер в Docker на том же ПК, клиент читает файл
./traker -ip 127.0.0.1 -p 8080 -v ../video/SkySun.mp4

# Сервер на Raspberry Pi в локальной сети
./traker -ip 192.168.1.50 -p 8080 -c
```

**Флаги:**

| Флаг | Значение |
|------|----------|
| `-ip <адрес>` | IP-адрес сервера |
| `-p <порт>` | Порт сервера |
| `-c` | Захват с камеры устройства |
| `-v <путь>` | Путь к видеофайлу (зацикливается по окончании) |

### Проверка сервера через curl

```bash
curl -X POST http://127.0.0.1:8080/commands \
  -H "Content-Type: application/json" \
  -d '{"command": "start"}'
```

## ⚙️ Принцип работы

1. **Детекция** — кадр переводится в HSV, накладывается маска на яркие области; солнце определяется как самый яркий контур вблизи центра кадра.
2. **Трекинг** — каждое облако сопровождается фильтром Калмана (состояние: x, y, vx, vy, w, h). Ассоциация детекций с треками — по метрике IoU.
3. **Прогноз** — для каждого зрелого трека (возраст ≥ 4 кадров) вычисляется время до точки максимального сближения с солнцем. Если `t < 3 сек` — угроза перекрытия.
4. **Команда** — клиент отправляет `POST /commands {"command": "start"}` при угрозе и `{"command": "stop"}` когда солнце свободно.
5. **Исполнение** — сервер обрабатывает команду через `EngineStateMachine` и переключает GPIO-пины мотора.

## 📝 Заметки

- **FSM** игнорирует дублирующие команды: повторный `start` при уже работающем моторе или `stop` при уже остановленном — безопасно пропускается.
- **GPIO** использует `ioctl` с `GPIOHANDLE_REQUEST_OUTPUT`. На реальном устройстве убедись, что пользователь имеет доступ к `/dev/gpiochip*` (группа `gpio` или `sudo`).
- При тестировании на ПК/Docker ошибки `No such file or directory` для `/dev/gpiochip0` — ожидаемы: GPIO существует только на реальном железе, FSM при этом работает корректно.
